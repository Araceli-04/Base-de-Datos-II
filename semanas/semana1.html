01

Arquitecturas de Datos

La arquitectura de una base de datos es la columna vertebral técnica que define cómo una aplicación se comportará bajo presión. Su diseño debe considerar tres pilares fundamentales: Eficiencia, Escalabilidad y Seguridad. En esta fase, el uso del esquema ANSI-SPARC permite separar la interacción del usuario de la estructura física del almacenamiento.

🏛️ Centralizada

Un modelo robusto donde un único nodo procesa todas las transacciones, facilitando el control total.

🖥️ Cliente-Servidor

Distribución de tareas donde el cliente solicita información y el servidor especializado responde con datos.

🌐 Distribuida

Bases de datos integradas lógicamente pero situadas en nodos físicos distintos para asegurar alta disponibilidad.

☁️ Nube

Infraestructura virtualizada que ofrece flexibilidad total y acceso remoto seguro bajo demanda.

02

Sistemas de Repositorios

📊 Data Warehouse

Almacén optimizado para el análisis histórico. Utiliza datos estructurados para decisiones estratégicas.

🌊 Data Lake

Repositorio masivo que conserva los datos en su estado nativo, permitiendo una analítica flexible.
